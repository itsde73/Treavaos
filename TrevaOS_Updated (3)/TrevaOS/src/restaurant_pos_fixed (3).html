🍽 MENU STRUCTURE
Categories: Soups · Starters · Mains · Biryani · Breads · Desserts · Beverages · Bar
Section routing: Each item tagged to kitchen section (Indian / Chinese / Pizza etc.) or Bar
Item attributes: Veg 🟢 / Non-veg 🔴 / Egg 🟡 / Vegan / Jain
Allergen tags: Gluten / Nuts / Dairy / Shellfish etc.
Course flag: Starter / Main / Dessert / Drinks
Variants: Half / Full / Small / Medium / Large
Modifiers: Add-ons / Remove / Extra / Spice level
Availability: Lunch only / Dinner only / All day / Seasonal
Status: Active / 86'd (live) / Archived / Draft
📋 RECIPE MASTER
Per dish: Step-by-step method, plating guide, reference photo
Yield: Recipe qty → no. of portions output
Portion size: Grams / ml per serving
Prep time: Target in minutes — feeds KOT timer
Cooking method: Tandoor / Grill / Fried / Steamed / Raw
Station: Which kitchen section handles this dish
Versions: Recipe versioning — chef can update, old version archived
Chef access: View & edit; Owner / Manager approves publish
Recipe change triggers raw material & cost recalculation
🥕 RAW MATERIAL MAPPING
Each dish ↔ ingredients: exact qty per serving linked to Main Store items
Unit mapping: Store unit (kg) → Recipe unit (gm) conversion handled auto
KOT fires → deduction: recipe qty × covers auto-deducted from section stock
Substitute ingredient: if primary OOS, system suggests mapped substitute
Wastage factor: configured per ingredient (e.g. onion 10% peeling loss)
Semi-processed items: e.g. "Gravy Base" can itself be a recipe with raw materials
Bar mapping: Cocktail → exact ml of each spirit + mixer + garnish
Mapping change auto-recalculates food cost % per dish
💰 PRICING & APPROVAL FLOW
Chef proposes: dish added with cost, suggested margin — no price set
Manager sets price: reviews food cost %, applies desired margin → sets MRP
Owner approves: price change above threshold → Owner OTP / PIN required
GST rate: Owner assigns correct slab (5% / 12% / 18%) per item
Combo pricing: Manager creates bundle; cost vs. saving shown
Happy Hour: time-based price rules configured by Manager
Price history: all changes logged with who changed it and when
Aggregator price: can be different from dine-in price — managed separately
Published price auto-syncs to QR menu, POS, aggregators
